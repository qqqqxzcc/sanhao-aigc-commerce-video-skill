# 【三好AIGC】AI批量产出带货视频Skill

这个 skill 用于抖音/短视频带货视频生产。后续与用户沟通时，统一使用以下命名：`功能1`、`功能2`、`功能3`。其中 `功能2` 下面再细分为 `路线1`、`路线2`、`路线3`、`路线4`。

## 功能选择

开始前，必须先根据用户输入识别并只选择一个功能或一条路线执行，不要混用。

- `功能1`：对标视频复刻。用户提供 benchmark/竞品视频、产品图、模特图、卖点和生成数量时，你先拆解原视频，先生成 1 张复刻分镜图给用户确认，再按用户要的数量继续生成剩余分镜图，最后再生成对应的 `Seedance 2.0` 视频提示词，并可继续调用视频 API。
- `功能2`：强hook 素材生视频链路。`功能2` 内部必须继续识别 `路线1`、`路线2`、`路线3`、`路线4`。
- `功能3`：用户已经提供现成分镜图和对应提示词，你跳过脚本和出图阶段，直接整理生成 Seedance 2.0 视频提示词，并可选继续调用视频 API。
- 如果用户只说“改功能2”或“走路线1/路线2”，默认进入 `功能2` 并按路线执行。
- 如果用户同时给出多种信号，但没有明确指定功能或路线，先询问用户要走哪一个。
- 同一轮运行只执行一个功能；`功能2` 同一轮只执行一条路线。

## 共用规则

- 产品一致性和模特一致性是 P0 规则。只要产品变形、换款、换色、Logo 错位、模特变脸、发型服装明显漂移，这一轮就算失败。
- 如果用户给的是原始产品图而不是白底三视图，先自动生成白底三视图，作为内部 product-lock 参考素材。除非用户要求看，或出现严重保真问题，否则不必主动展示。
- 白底三视图必须尽量锁定：产品形状、结构、颜色、材质、纹理、Logo/文字位置、包装细节。
- 如果用户提供了产品卖点文字，直接使用；如果用户没给文字卖点，但给了卖点图片、详情页截图、海报图或其他视觉素材，必须先提取并整理卖点，再展示给用户确认后继续。
- 如果 benchmark 视频里存在口播、人物对话、字幕承载的文案信息或明显说话场景，绝对不能忽略，必须先拆出原视频的对话/口播结构，再用用户产品卖点进行对应替换。
- 生成分镜脚本时，必须清楚标序号。生成分镜图时，也必须清楚标序号。后续提示词也要能一一对应到序号。
- 默认交付形式必须是按编号拆开的单独 9 宫格分镜图文件，例如 `分镜图1`、`分镜图2`、`分镜图3`。默认规则是 `1条脚本 = 1张独立9宫格分镜图`。除非用户明确要求把多条脚本拼成一张总图，否则不要擅自合并成交联系表、总故事板或多脚本拼版图。
- 所有后续视频生成链路，默认都要保持产品不变、模特不变。无论用户要 1 条还是多条，都不能因为数量增加而漂移人物或产品身份。
- 只要用户上传的素材或你生成的参考资产已经可以用于后续步骤，就要尽快转成公开 URL，并保存 URL 清单，方便后续调用视频 API。
- 当目标视频 API 支持本地文件、data URL、公开 URL 时，默认优先使用公开 URL。
- 如果要求口播：口播文案与逐镜头口播分配
- 最终视频提示词默认面向 `Seedance 2.0` 编写，语言要具体、导演化、可执行。
- 只要用户明确写了 `是否要口播：要`，就必须把“有口播”视为全链路强约束：分镜脚本必须包含说话场景，分镜图/故事板必须支持人物正在说话，最终交付必须补齐口播文案、逐镜头口播分配和带口播约束的 Seedance 2.0 提示词。
- 只要用户明确写了“不要字幕”或同义要求，就必须把“无字幕”视为全链路强约束：分镜图、故事板、视频提示词、API 提交要求里都不能出现字幕、贴纸字卡或自动字幕。
- 生成 Seedance 2.0 提示词时，必须明确要求这是连续的真人动态视频，而不是静态分镜板动起来。提示词必须显式约束：`American English spoken dialogue`、`visible speaking mouth movement / lip sync`、`continuous live-action video`、`natural body motion`、`hand motion`、`eye movement`、`camera drift`、`action progression`，并明确禁止 `static collage`、`slideshow`、`repeated still frames`、`frozen posing`、`storyboard board coming alive` 这类结果。
- 在进入“生视频”这一步之前，必须询问用户采用哪种方式：
  - `方式A`：由你直接调用 API 生成视频
  - `方式B`：由用户自己拿提示词和分镜图去网站生成视频
- 如果用户一开始就明确给出 `生成数量：N`，你必须严格按这个数量执行；如果当前路线中还有一个需要单独确认的“分镜图数量”没有给出，必须先问清楚。

## 功能1：对标视频复刻

适用场景：用户提供 benchmark/竞品视频，想按原视频结构复刻带货视频。

### 默认输入

- benchmark 视频
- 产品信息
- 产品图片
- 模特参考图
- 卖点文字或卖点图片
- `生成数量：N`
- 可选已有分镜图

### 工作流

1. 读取并校验输入：benchmark 视频、产品图、模特图、卖点素材、`生成数量：N`。如果 `N` 没给，必须先询问数量再继续。
2. 拆解原视频的镜头顺序、动作、节奏、场景、文案结构，以及所有可见口播/对话/说话场景。只要原视频里有人物在说话、配音、字幕承载口播信息或明显对话节奏，就必须把这部分拆出来，不得忽略。
3. 如果用户给的是卖点文字，直接整理成可用卖点清单；如果用户给的是卖点图片、详情页截图、海报图或其他视觉素材，先提取卖点并整理给用户确认。未确认卖点前，不要继续写替换文案。
4. 基于 benchmark 视频的对话/口播结构，用用户确认后的卖点替换成用户产品版本的文案。替换后必须保持原视频的镜头顺序、说话节奏、场景逻辑和卖点推进逻辑，但内容换成用户产品。
5. 如果没有白底三视图，先自动生成内部 product-lock 三视图。
6. 先只生成 `1张` 复刻分镜图，作为打样图给用户确认。这个阶段必须已经把用户产品和用户模特替换进去，并把产品一致性、模特一致性当成 P0 硬约束。
7. 输出这 `1张` 复刻分镜图时，必须清晰标注为 `分镜图1（打样确认）`，并停下来等待用户确认或修改意见。未确认前，不要继续批量生成后续分镜图。
8. 在用户确认 `分镜图1` 后，严格按用户给定的 `生成数量：N` 继续生成剩余分镜图。默认规则是总共输出 `N` 张独立编号分镜图；如果 `N=1`，则打样确认通过后直接进入提示词阶段。
9. 输出多张分镜图时，必须清晰标序号，例如 `分镜图1`、`分镜图2`、`分镜图3`。无论生成多少张，产品和模特都必须保持同一身份，不允许因为版本增加而换产品、换脸、换发型、换服装。
10. 分镜图全部生成后，必须再次停下来给用户确认。用户确认前，不要进入 `Seedance 2.0` 提示词阶段。
11. 在用户确认分镜图后，根据每一张分镜图分别生成适用于 `Seedance 2.0` 的视频提示词。如果 benchmark 视频存在对话、口播、说话动作或说话节奏，最终提示词里必须保留“人物正在说话/有口播/嘴型同步/说话节奏对应”的硬约束，绝不能忽略。
12. 如果当前内容包含口播或对话，再补齐与每张分镜图一一对应的口播文案和逐镜头口播分配。
13. 把白底三视图、产品图、模特图、分镜图等参考资产转成公开 URL，并保存 URL 清单，便于后续 API 调用。
14. 在进入“生视频”前，必须询问用户采用哪种方式：
   - `方式A`：由你调用 API 生成视频
   - `方式B`：由用户自己拿提示词和分镜图去网站生成视频
15. 如果用户选择 `方式A`，必须先向用户索要相关 API 信息或密钥，再继续调用 API 生成视频；如果用户选择 `方式B`，只交付分镜图、口播文案、逐镜头口播分配、Seedance 2.0 提示词和 URL 清单。

### 最终交付

- 原视频拆解结果（包含镜头结构和可见对话/口播结构）
- 用户卖点整理结果；如果卖点来自图片提取，必须先有确认版
- 替换后的产品卖点文案/口播文案
- 带序号的独立分镜图文件
- 如果存在口播或对话：与分镜图一一对应的口播文案与逐镜头口播分配
- 与分镜图一一对应的 `Seedance 2.0` 视频提示词
- 公开 URL 清单
- 如果用户选择 `方式A`：生成后的视频文件和 QC 结果

## 功能2：强hook 素材生视频链路

`功能2` 下面必须按 `路线1`、`路线2`、`路线3`、`路线4` 继续区分。

### 路线1

适用场景：用户希望你根据上传素材，先生成指定条数的分镜脚本，再一一生成对应分镜图，再生成对应的 Seedance 2.0 视频提示词，最后调用api或手动生成视频。

#### 路线1输入

- 用户上传的产品素材
- 可选模特素材
- 可选卖点、受众、语气、场景要求
- `生成数量：N`
- 可选语言、时长、字幕要求、音频要求

#### 路线1工作流

1. 读取用户素材，并确认或提取 `生成数量：N`。如果用户没有给出数量，先询问数量，再继续。
2. 如果没有白底三视图，先自动生成内部 product-lock 三视图。
3. 基于素材生成 `N` 条分镜脚本。
4. 输出分镜脚本时，必须用清晰序号标注，例如 `脚本1`、`脚本2`、`脚本3`。这一步完成后必须停下来给用户过目，等待确认或修改意见。
5. 在用户确认分镜脚本后，按脚本一一生成对应的 `N` 张分镜图。默认每条脚本输出 1 张独立的 9 宫格分镜图文件。
6. 输出分镜图时，必须用与脚本一致的清晰序号标注，例如 `分镜图1` 对应 `脚本1`。默认逐张交付独立的 9 宫格分镜图文件。这一步完成后也必须停下来给用户确认。
7. 在用户确认分镜图后，先为每一条内容分别生成完整口播文案和逐镜头口播分配（如果用户要求口播）。
8. 再为每一条分镜图分别生成适用于 `Seedance 2.0` 的视频提示词，并把“有口播、嘴型同步、连续动态视频、非静态拼图感、无字幕”等要求写成硬约束。
9. 生成多条提示词时，必须显式遵守“产品不变、模特不变”规则，不能因为有多个版本而换产品或换模特。
11. 在脚本、分镜图或其他参考资产一旦确认可用于后续步骤时，就把它们转成公开 URL，并保存可复用的 URL 清单。
12. 进入生视频前，必须询问用户采用 `方式A` 还是 `方式B`。
12. 如果用户选择 `方式A`，再继续调用 API 生成视频；如果选择 `方式B`，只交付分镜脚本、分镜图、口播文案、逐镜头口播分配、提示词和 URL 清单。

#### 路线1最终交付

- 带序号的分镜脚本
- 带序号、按编号拆开的独立 9 宫格分镜图文件
- 如果要求口播：与序号一一对应的口播文案与逐镜头口播分配
- 与序号一一对应的 Seedance 2.0 视频提示词
- 参考素材公开 URL 清单
- 如果用户选择 `方式A`：生成后的视频文件和 QC 结果

### 路线2

适用场景：用户希望你先生成 `1个` 分镜脚本，确认后，再围绕这同一个脚本生成用户一开始就指定数量的多张分镜图，然后生成对应的 Seedance 2.0 视频提示词，最后调用api或手动生成视频。

#### 路线2输入

- 用户上传的产品素材
- 可选模特素材
- 可选卖点、受众、语气、场景要求
- 分镜图数量；优先读取用户一开始给出的数量，如果没给，必须询问
- 可选语言、时长、字幕要求、音频要求

#### 路线2工作流

1. 读取用户素材。如果没有白底三视图，先自动生成内部 product-lock 三视图。
2. 先生成 `1个` 分镜脚本。
3. 把这个分镜脚本单独输出给用户确认；未确认前，不要继续生成分镜图或视频提示词。
4. 在脚本确认后，读取并使用用户一开始给出的分镜图数量；如果用户还没有给出数量，必须先询问清楚再继续。
5. 围绕同一个分镜脚本，生成用户指定数量的多张分镜图。默认每一张都输出为 1 张独立的 9 宫格分镜图文件。
6. 这些分镜图必须清晰标序号，例如 `分镜图1`、`分镜图2`、`分镜图3`，并逐张交付独立的 9 宫格分镜图文件。它们都对应同一个已确认的分镜脚本。
7. 即使基于同一个脚本生成多张分镜图，也必须保持产品不变、模特不变。允许变化的是镜头表现、动作细节、场景细节、构图变化，但不能换人换货。
8. 分镜图生成后，先给用户确认；如果用户要求挑图或删图，先处理确认，再进入提示词阶段。
9. 根据确认后的这些分镜图，先分别生成对应的口播文案与逐镜头口播分配（如果用户要求口播）。
10. 再分别生成对应的 `Seedance 2.0` 视频提示词，并与序号一一对应，同时显式要求口播嘴型、连续动态视频、非静态拼图感和无字幕。
11. 在脚本、分镜图或其他参考资产一旦确认可用于后续步骤时，就把它们转成公开 URL，并保存可复用的 URL 清单。
12. 进入生视频前，必须询问用户采用 `方式A` 还是 `方式B`。
13. 如果用户选择 `方式A`，再继续调用 API 生成视频；如果选择 `方式B`，只交付分镜脚本、分镜图、口播文案、逐镜头口播分配、提示词和 URL 清单。

#### 路线2最终交付

- 已确认的单个分镜脚本
- 带序号、按编号拆开的独立 9 宫格分镜图文件（全部对应同一个已确认脚本）
- 如果要求口播：与序号一一对应的口播文案与逐镜头口播分配
- 与序号一一对应的 Seedance 2.0 视频提示词
- 参考素材公开 URL 清单
- 如果用户选择 `方式A`：生成后的视频文件和 QC 结果

### 路线3

适用场景：用户提供产品、模特等素材，并在一开始给出 `生成数量：N`。你先生成 `N` 条分镜脚本，确认后，再先为每条脚本生成 1 张独立 9 宫格视觉母版，然后基于每张视觉母版继续生成 1 张更完整的 16:9 故事板，最后生成对应的 Seedance 2.0 视频提示词，最后调用api或手动生成视频。

#### 路线3输入

- 用户上传的产品素材
- 可选模特素材
- 可选卖点、受众、语气、场景要求
- `生成数量：N`
- 可选语言、时长、字幕要求、音频要求

#### 路线3工作流

1. 读取用户素材，并确认或提取 `生成数量：N`。如果用户没有给出数量，先询问数量，再继续。
2. 如果没有白底三视图，先自动生成内部 product-lock 三视图。
3. 基于素材生成 `N` 条分镜脚本。
4. 输出分镜脚本时，必须清晰标注序号，例如 `脚本1`、`脚本2`、`脚本3`。这一步完成后必须停下来给用户确认。
5. 在用户确认分镜脚本后，先为每条脚本分别生成 1 张独立的 9 宫格视觉母版。每张视觉母版都必须与对应脚本一一对应，并保持产品和模特不变。
6. 输出 9 宫格视觉母版时，必须清晰标注序号，例如 `分镜图1` 对应 `脚本1`。这一步完成后也必须停下来给用户确认。
7. 在用户确认 9 宫格视觉母版后，基于每张视觉母版继续生成 1 张更完整的 16:9 故事板。
8. 这张 16:9 故事板不是简单拼图，而是更接近真实拍摄方案的制作板：镜头细节、动作信息、景别、构图和拍摄意图可以更完整，但整体信息不能杂乱，目标是让后续生成视频更加精细。
9. 输出 16:9 故事板时，必须与脚本序号、9 宫格视觉母版序号一一对应，并停下来给用户确认。
10. 在用户确认 16:9 故事板后，先为每一条内容分别生成完整口播文案与逐镜头口播分配（如果用户要求口播）。
11. 再为每一条内容分别生成适用于 `Seedance 2.0` 的视频提示词，并显式要求口播嘴型、连续动态视频、非静态拼图感和无字幕。
12. 无论生成多少条，都必须显式遵守“产品不变、模特不变”规则，不能因为版本变多而换产品或换模特。
13. 在脚本、9 宫格视觉母版、16:9 故事板或其他参考资产一旦确认可用于后续步骤时，就把它们转成公开 URL，并保存可复用的 URL 清单。
14. 进入生视频前，必须询问用户采用 `方式A` 还是 `方式B`。
15. 如果用户选择 `方式A`，再继续调用 API 生成视频；如果用户选择 `方式B`，只交付分镜脚本、9 宫格视觉母版、16:9 故事板、口播文案、逐镜头口播分配、提示词和 URL 清单。

#### 路线3最终交付

- 带序号的分镜脚本
- 带序号、按编号拆开的独立 9 宫格视觉母版文件
- 带序号、与脚本一一对应的独立 16:9 故事板文件
- 如果要求口播：与序号一一对应的口播文案与逐镜头口播分配
- 与序号一一对应的 Seedance 2.0 视频提示词
- 参考素材公开 URL 清单
- 如果用户选择 `方式A`：生成后的视频文件和 QC 结果
### 路线4

适用场景：用户提供产品、模特等素材，并在一开始给出 `生成数量：N`。你先只生成 `1个` 分镜脚本，确认后，再围绕这同一个脚本生成用户指定数量的独立 9 宫格视觉母版，再基于这些视觉母版分别生成对应数量的更完整的 16:9 故事板，最后生成对应的 Seedance 2.0 视频提示词，最后调用api或手动生成视频。

#### 路线4输入

- 用户上传的产品素材
- 可选模特素材
- 可选卖点、受众、语气、场景要求
- `生成数量：N`
- 可选语言、时长、字幕要求、音频要求

#### 路线4工作流

1. 读取用户素材，并确认或提取 `生成数量：N`。如果用户没有给出数量，先询问数量，再继续。
2. 如果没有白底三视图，先自动生成内部 product-lock 三视图。
3. 基于素材先生成 `1个` 分镜脚本。
4. 输出这个分镜脚本时，必须单独展示并清晰标注，等待用户确认；未确认前，不要继续生成视觉母版、故事板或视频提示词。
5. 在用户确认这 `1个` 分镜脚本后，围绕该脚本生成用户指定数量的独立 9 宫格视觉母版。默认规则是 `1个脚本 + N张独立9宫格视觉母版`。
6. 这些 9 宫格视觉母版必须清晰标注序号，例如 `分镜图1`、`分镜图2`、`分镜图3`，并逐张交付。它们都对应同一个已确认脚本。
7. 即使基于同一个脚本生成多张 9 宫格视觉母版，也必须保持产品不变、模特不变。允许变化的是镜头表现、动作细节、场景细节、构图变化，但不能换人换货。
8. 9 宫格视觉母版生成后，先给用户确认；如果用户要求挑图、删图或改图，先处理确认，再进入故事板阶段。
9. 在用户确认这些 9 宫格视觉母版后，基于每一张视觉母版分别生成对应数量的独立 16:9 故事板。
10. 这些 16:9 故事板不是简单拼图，而是更接近真实拍摄方案的制作板：要嵌入对应的 9 宫格视觉母版，补充更完整的镜头细节、动作信息、景别、构图、人物状态、产品锁定、拍摄意图等内容，但整体信息不能杂乱，目标是让后续生成视频更加精细。
11. 输出 16:9 故事板时，必须与脚本序号、9 宫格视觉母版序号一一对应，并停下来给用户确认。
12. 在用户确认 16:9 故事板后，先为每一条内容分别生成完整口播文案与逐镜头口播分配（如果用户要求口播）。
13. 再为每一条内容分别生成适用于 `Seedance 2.0` 的视频提示词，并显式要求口播嘴型、连续动态视频、非静态拼图感和无字幕。
14. 在脚本、9 宫格视觉母版、16:9 故事板或其他参考资产一旦确认可用于后续步骤时，就把它们转成公开 URL，并保存可复用的 URL 清单。
15. 进入生视频前，必须询问用户采用 `方式A` 还是 `方式B`。
16. 如果用户选择 `方式A`，再继续调用 API 生成视频；如果用户选择 `方式B`，只交付分镜脚本、9 宫格视觉母版、16:9 故事板、口播文案、逐镜头口播分配、提示词和 URL 清单。

#### 路线4最终交付

- 已确认的单个分镜脚本
- 带序号、按编号拆开的独立 9 宫格视觉母版文件（全部对应同一个已确认脚本）
- 带序号、与脚本一一对应的独立 16:9 故事板文件
- 如果要求口播：与序号一一对应的口播文案与逐镜头口播分配
- 与序号一一对应的 Seedance 2.0 视频提示词
- 参考素材公开 URL 清单
- 如果用户选择 `方式A`：生成后的视频文件和 QC 结果

## 功能3：已有分镜图与提示词直转视频

适用场景：用户已经有分镜图和对应提示词，希望直接整理成最终 Seedance 2.0 视频提示词，或继续调用 API。

### 默认输入

- 分镜图
- 对应提示词
- 可选产品图
- 可选模特图
- 可选时长、字幕、音频要求

### 工作流

1. 校验用户提供的分镜图和提示词是否齐全。
2. 如果需要，用产品图和模特图补足 product-lock 参考。
3. 把参考素材转成公开 URL，并保存 URL 清单。
4. 如果用户要求口播，先补齐完整口播文案与逐镜头口播分配。
5. 生成适用于 `Seedance 2.0` 的最终视频提示词，并显式要求口播嘴型、连续动态视频、非静态拼图感和无字幕。
6. 询问用户采用 `方式A` 还是 `方式B` 生视频。
7. 如果用户选 `方式A`，再继续调用 API；如果用户选 `方式B`，只交付口播文案、逐镜头口播分配、提示词、参考图和 URL 清单。

### 最终交付

- 如果要求口播：口播文案与逐镜头口播分配
- 最终视频提示词
- 参考素材公开 URL 清单
- 如果用户选择 `方式A`：生成后的视频文件和 QC 结果

## 质量门槛

- 任何一步只要出现产品换款、换色、变形、Logo 漂移，就必须修正后再继续。
- 任何一步只要出现模特身份明显变化，也必须修正后再继续。
- 分镜脚本、分镜图、视频提示词三者必须能按序号一一对应。只要用户要求口播，口播文案与逐镜头口播分配也必须与同一序号一一对应。
- `功能2` 的 `路线1` 与 `路线2` 都必须包含两次用户确认：
  - 脚本确认
  - 分镜图确认
- 进入“生视频”前，必须明确用户采用 `方式A` 还是 `方式B`。
- `功能1` 必须包含两次用户确认：
  - `分镜图1` 打样确认
  - 全部分镜图确认
- 只要用户要求口播，最终交付的脚本、分镜图、故事板和 Seedance 提示词里都必须体现“人物正在说话”，不能漏掉口播要求。
- 只要 benchmark 视频本身存在对话、口播、配音或说话镜头，即使用户没有单独强调“要口播”，也必须在拆解、文案替换和最终提示词里保留这部分说话信息，不能当成纯静态展示视频处理。
- 只要用户要求无字幕，最终交付的分镜图、故事板、提示词和 API 提交要求里都不能出现字幕或任何字卡。
- 进入生视频前必须做一次动态视频 QA：确认提示词没有把结果导向静态拼图、重复帧、僵硬站桩、只有产品展示没有说话动作的失败形态。
- 需要调用视频 API 时，必须优先使用整理好的公开 URL 参考素材。

## 输出风格

- 回复要短、直接、可执行。
- 给用户看脚本或分镜图时，必须标序号，方便确认。
- 默认直接给按编号拆开的独立 9 宫格分镜图，不要擅自改成多脚本拼版总图或整张总故事板。
- 用户要求改某一条时，直接按序号定位，不要重排全量内容，除非用户要求整体重做。
- 用户要求改提示词时，输出完整替换版，不要只给 diff。
